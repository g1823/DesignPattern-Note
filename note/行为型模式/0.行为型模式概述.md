# 行为型模式（Behavioral Patterns）

## 一、定义与核心思想

行为型模式关注的是**对象之间的职责分配与通信方式**。
 它主要解决的是：在复杂系统中，**如何让对象之间高效协作，同时降低耦合度，使行为更灵活、更可扩展**。

> 🌱 核心目标：
>
> - 解耦对象之间的交互关系
> - 明确职责分配（谁做什么）
> - 提升系统的扩展性与可维护性
> - 支持行为的动态变化
> - 简化复杂流程控制

------

## 二、模式分类概览

| 模式名称     | 主要目的       | 关键思想                       | 典型应用               |
| ------------ | -------------- | ------------------------------ | ---------------------- |
| 职责链模式   | 请求逐级处理   | 请求沿处理链传递，直到被处理   | 过滤器链、审批流       |
| 命令模式     | 请求封装       | 将请求封装为对象，支持解耦调用 | 按钮操作、任务队列     |
| 解释器模式   | 语法解析       | 定义语言文法并解释执行         | 表达式解析、规则引擎   |
| 迭代器模式   | 统一遍历       | 提供统一访问集合的方式         | 集合遍历（Iterator）   |
| 中介者模式   | 中央协调通信   | 用中介对象封装复杂对象交互     | 聊天室、调度中心       |
| 备忘录模式   | 状态保存与恢复 | 保存对象状态用于回滚           | 撤销操作、快照机制     |
| 观察者模式   | 事件通知       | 一对多依赖，自动通知更新       | 事件监听、消息订阅     |
| 状态模式     | 状态驱动行为   | 状态变化驱动行为变化           | 订单状态流转、状态机   |
| 策略模式     | 算法切换       | 封装算法，运行时动态替换       | 支付策略、折扣策略     |
| 模板方法模式 | 固定流程扩展   | 定义算法骨架，子类实现细节     | Spring模板类、JDBC模板 |
| 访问者模式   | 操作分离       | 将数据结构与操作解耦           | 编译器、复杂结构处理   |

## 三、行为型模式核心原则

1. **单一职责原则（SRP）**
    → 每个对象专注于自己的行为（如策略、命令、状态）。
2. **开闭原则（OCP）**
    → 新行为通过扩展实现，而不是修改已有代码（策略、访问者）。
3. **面向接口编程**
    → 行为抽象为接口（如策略接口、观察者接口）。
4. **低耦合通信**
    → 通过职责链、观察者、中介者降低对象间依赖。
5. **行为封装与解耦**
    → 将“请求/行为”和“执行者”分离（命令、策略）。

## 四、行为型模式应用场景对比

| 模式   | 动机         | 是否改变行为 | 是否解耦调用方 | 典型特征    |
| ------ | ------------ | ------------ | -------------- | ----------- |
| 职责链 | 多级处理     | ❌ 不变       | ✅              | 链式传递    |
| 命令   | 请求解耦     | ❌ 不变       | ✅              | 请求对象化  |
| 解释器 | 规则执行     | ✅ 改变       | ❌              | 语法解析    |
| 迭代器 | 统一遍历     | ❌ 不变       | ✅              | 遍历抽象    |
| 中介者 | 解耦通信     | ❌ 不变       | ✅              | 中央协调    |
| 备忘录 | 状态恢复     | ❌ 不变       | ❌              | 快照回滚    |
| 观察者 | 事件通知     | ❌ 不变       | ✅              | 发布订阅    |
| 状态   | 状态驱动行为 | ✅ 改变       | ❌              | 状态机      |
| 策略   | 替换算法     | ✅ 改变       | ✅              | 消除if-else |
| 模板   | 固定流程     | ❌ 不变骨架   | ❌              | 流程复用    |
| 访问者 | 操作扩展     | ✅ 新行为     | ✅              | 双重分派    |
